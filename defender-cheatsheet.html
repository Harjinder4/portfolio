Windows Defender — Control Cheat Sheet

Real-time monitoring · Sample submission · Tamper protection · OpSec

Windows 10/11 · Admin required

Disable

OFF
Real-time monitoring CMD
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Real-Time Protection" /v DisableRealtimeMonitoring /t REG_DWORD /d 1 /f

Sets registry value to 1 — stops active file scanning.

Sample submission CMD
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Spynet" /v SubmitSamplesConsent /t REG_DWORD /d 2 /f

Value 2 = NeverSend. Stops files being uploaded to Microsoft.

Tamper protection CMD
reg add "HKLM\SOFTWARE\Microsoft\Windows Defender\Features" /v TamperProtection /t REG_DWORD /d 4 /f

Value 4 = disabled. Must turn off in Security UI first if blocked.

Real-time monitoring PS
Set-MpPreference -DisableRealtimeMonitoring $true

PowerShell native cmdlet. Simpler and easier to script.

Sample submission PS
Set-MpPreference -SubmitSamplesConsent NeverSend

Cleaner syntax; same effect as registry value 2.

Tamper protection PS
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows Defender\Features" -Name "TamperProtection" -Value 4

Registry write via PowerShell. Same key as CMD version.

Combined — disable all at once

CMD Paste into an elevated Command Prompt 
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Real-Time Protection" /v DisableRealtimeMonitoring /t REG_DWORD /d 1 /f && reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Spynet" /v SubmitSamplesConsent /t REG_DWORD /d 2 /f && reg add "HKLM\SOFTWARE\Microsoft\Windows Defender\Features" /v TamperProtection /t REG_DWORD /d 4 /f
PS Paste into an elevated PowerShell window 
Set-MpPreference -DisableRealtimeMonitoring $true; Set-MpPreference -SubmitSamplesConsent NeverSend; Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows Defender\Features" -Name "TamperProtection" -Value 5

Re-enable

ON
Real-time monitoring CMD
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Real-Time Protection" /v DisableRealtimeMonitoring /t REG_DWORD /d 0 /f

Flip value back to 0 to restore scanning.

Sample submission CMD
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Spynet" /v SubmitSamplesConsent /t REG_DWORD /d 1 /f

Value 1 = SendSafeSamples (default behaviour).

Tamper protection CMD
reg add "HKLM\SOFTWARE\Microsoft\Windows Defender\Features" /v TamperProtection /t REG_DWORD /d 5 /f

Value 5 = enabled. Defender will re-lock itself.

Real-time monitoring PS
Set-MpPreference -DisableRealtimeMonitoring $false

Boolean false restores default on state.

Sample submission PS
Set-MpPreference -SubmitSamplesConsent SendSafeSamples

Restores automatic submission of safe samples.

Tamper protection PS
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows Defender\Features" -Name "TamperProtection" -Value 5

Re-enables tamper protection via registry.

Registry value reference

Setting Value Meaning Registry key
DisableRealtimeMonitoring 0 Enabled (scanning ON) …\Real-Time Protection
DisableRealtimeMonitoring 1 Disabled (scanning OFF) …\Real-Time Protection
SubmitSamplesConsent 0 Always prompt user …\Spynet
SubmitSamplesConsent 1 Send safe samples (default) …\Spynet
SubmitSamplesConsent 2 Never send anything …\Spynet
SubmitSamplesConsent 3 Send all samples silently …\Spynet
TamperProtection 4 Tamper protection OFF …\Windows Defender\Features
TamperProtection 5 Tamper protection ON (default) …\Windows Defender\Features

OpSec & general tips

01

Tamper protection blocks everything

Registry writes and Set-MpPreference will silently fail or be rolled back while tamper protection is active. Always disable it first — either from Windows Security UI (Settings → Virus & threat protection → Manage settings) or via the registry key, if already unlocked.

02

Always run as Administrator

Every command here requires an elevated session. If you get "Access denied" or the setting reverts instantly, you are not running as admin. Right-click CMD or PowerShell → "Run as administrator" before pasting.

03

Windows Update can re-enable everything

A Windows Defender definition or platform update can silently reset your registry values and re-enable tamper protection. After any update, verify your settings with Get-MpPreference in PowerShell.

04

Policy path vs non-policy path

The …\Policies\Microsoft\Windows Defender path takes precedence over …\Microsoft\Windows Defender (non-policy). If changes aren't sticking, check both keys — a policy key override may be winning.

05

Verify with Get-MpPreference

Run Get-MpPreference | Select DisableRealtimeMonitoring, SubmitSamplesConsent in PowerShell after making changes to confirm they actually applied and weren't silently rejected.

06

Sample submission ≠ cloud protection

Disabling sample submission stops file uploads, but cloud-delivered protection (MAPS) is a separate toggle: Set-MpPreference -MAPSReporting Disabled. Both need to be off to fully stop outbound telemetry to Microsoft.

07

Add exclusions instead of disabling

For most dev/testing scenarios, adding a folder exclusion is less risky than disabling Defender entirely: Add-MpPreference -ExclusionPath "C:\YourFolder". Leaves protection active everywhere else.

08

Domain-joined machines may be GPO-managed

On corporate or domain-joined machines, Group Policy can override every registry key here. Changes will apply but revert on the next GPO refresh cycle (every ~90 min). Check gpresult /h report.html to see applied policies.

Important warnings

⚠ Re-enable when done

Leaving Defender disabled is a real security risk. Run the re-enable commands as soon as your task is complete. Disable only for as long as you need.

Event log records changes

Every Defender state change is logged in Windows Event Viewer under Applications and Services → Microsoft → Windows → Windows Defender → Operational. The log can be cleared with: wevtutil cl "Microsoft-Windows-Windows Defender/Operational"

Check status at any time

Use Get-MpComputerStatus in PowerShell for a full status report including RealTimeProtectionEnabled, AntivirusEnabled, TamperProtectionSource, and more.

Bonus — useful diagnostic commands

Check full Defender status PS
Get-MpComputerStatus | Select RealTimeProtectionEnabled, AntivirusEnabled, TamperProtectionSource, AMRunningMode

Shows live state of all major protection features.

Check current preferences PS
Get-MpPreference | Select DisableRealtimeMonitoring, SubmitSamplesConsent, MAPSReporting, ExclusionPath

Verify your changes actually applied.

Add folder exclusion PS
Add-MpPreference -ExclusionPath "C:\Path\To\Folder"

Safer than full disable — only excludes a specific folder.

Disable cloud protection (MAPS) PS
Set-MpPreference -MAPSReporting Disabled

Stops cloud lookups separately from sample submission.
